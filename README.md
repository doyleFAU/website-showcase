# Vervio

**A free website planning app** — try fonts, colors, homepage layouts, and feature ideas in a live preview before you hire a designer or use a website builder.

**Live site:** https://website-showcase-lemon.vercel.app

Vervio is for **planning**, not publishing. Visitors explore styles, save a wish list, and optionally create an account to sync picks across devices. No install required.

---

## What you can do

| Area | What it offers |
|------|----------------|
| **Pick a Style** | 50 color themes, 32 fonts, light/dark mode — live preview on every page |
| **Home Pages** | 16 homepage layout presets (tech, bakery, portfolio, medical, and more) |
| **Quick Start** | Industry shortcuts that pre-fill colors, fonts, and add-ons |
| **Fonts & Colors** | Dedicated galleries to browse typography and palettes |
| **Add-ons** | 28-item wish list (contact form, booking, gallery, SEO, etc.) |
| **My List** | Summary of style picks + checked features |
| **Start Here** | 3-minute guided tutorial |
| **Accounts** | Optional login — saves picks to Supabase and syncs across devices |

### SEO & content pages

Built-in pages for search and local discovery:

- `faq.html` — FAQ with structured data
- `local.html` — South Florida / website planning local page
- `planning-guide.html` — How to plan a small business website
- `homepage-ideas.html` — Homepage layout inspiration

The build also generates `sitemap.xml`, `robots.txt`, and `llms.txt`.

## Quick start

```bash
git clone https://github.com/doyleFAU/website-showcase.git
cd website-showcase
cp .env.example .env
```

Edit `.env` with your Supabase keys (see [Supabase setup](#supabase-setup) below). Keys are never committed — they are injected at build time.

```bash
npm run build    # generate config + HTML pages
npm run serve    # preview at http://127.0.0.1:8090
```

Open http://127.0.0.1:8090 in your browser.

> **Important:** Do not open HTML files directly (`file://`). The app loads partials over HTTP and needs a simple web server.

If port 8090 is busy:

```bash
lsof -ti :8090 | xargs kill -9
npm run serve
```

### Editing content

1. **Page sections** — edit files in `partials/`, then run `npm run build:pages`
2. **Themes / fonts / add-ons** — `js/config/features.js`
3. **Default sandbox state** — `js/config/state.js`
4. **Styles** — `css/` (import order in `css/main.css`)

Generated `*.html` files at the repo root are build output. Prefer editing `partials/` and rebuilding so all pages stay in sync.

## How the app works

### Live style sandbox

`js/modules/sandbox.js` applies user picks to `<html>` via `data-theme`, `data-font`, `data-mode`, etc. CSS custom properties in `themes.css` and `fonts.css` update the whole page instantly.

Choices are saved to **localStorage** (`vervio-choices`, `vervio-features`) so they persist without an account.

### Optional cloud sync

When Supabase is configured and a user logs in:

1. `auth.js` loads or creates a row in `user_plans`
2. Changes debounce to cloud save (~900ms)
3. `choices` (jsonb) + `features` (jsonb) sync across devices

Works offline-first — local picks always apply; cloud sync is best-effort in the background.

## Supabase setup

### 1. Create a project

At [supabase.com](https://supabase.com), create a project and note the **Project URL** and **anon public** key.

### 2. Environment variables

**Local** — `.env`:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
```

**Production** — Vercel project settings → Environment Variables (same keys). Vercel runs `npm run build` on deploy, which generates `js/config/supabase.js`.

Optional build vars:

```env
SITE_URL=https://your-domain.com
GOOGLE_SITE_VERIFICATION=your-verification-code
```

### 3. Database table

Create `public.user_plans`:

| Column | Type | Notes |
|--------|------|-------|
| `id` | `uuid` | Primary key, default `gen_random_uuid()` |
| `user_id` | `uuid` | Required, **unique**, FK → `auth.users.id` (on delete cascade) |
| `choices` | `jsonb` | Theme, font, mode, toggles |
| `features` | `jsonb` | Array of add-on IDs |
| `updated_at` | `timestamptz` | Last save time |

Enable **RLS** and add policies for role `authenticated`:

```sql
create policy "Users read own plan"
  on public.user_plans for select to authenticated
  using (auth.uid() = user_id);

create policy "Users insert own plan"
  on public.user_plans for insert to authenticated
  with check (auth.uid() = user_id);

create policy "Users update own plan"
  on public.user_plans for update to authenticated
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);
```

### 4. Auth settings

In Supabase → **Authentication**:

- Enable **Email** provider
- **URL Configuration** — add your production URL and `/**` redirect pattern
- Include `https://your-site.com/reset-password.html` for password resets

## Deploy to Vercel

The repo is wired for Vercel static deploy:

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "."
}
```

**Deploy via Git** (recommended):

```bash
git push origin main
```

Vercel auto-builds on push. Set `SUPABASE_URL` and `SUPABASE_ANON_KEY` in the Vercel dashboard.

**Deploy via CLI:**

```bash
npx vercel --prod
```

### Security headers

`vercel.json` sets CSP, HSTS, `X-Frame-Options`, and related headers. Supabase and Google Fonts are allowlisted in CSP.

---

## Security notes

- Only the **anon** key belongs in the frontend — never commit `service_role` or Stripe secrets
- `.env` and generated `js/config/supabase.js` are gitignored locally; production keys live in Vercel env vars
- Row Level Security restricts `user_plans` to `auth.uid() = user_id`
- User input is sanitized in `js/modules/security.js` before save/display

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Page looks broken / no header | Use `npm run serve`, not `file://` |
| `Address already in use` on serve | `lsof -ti :8090 \| xargs kill -9` then retry |
| Login disabled | Set `SUPABASE_URL` + `SUPABASE_ANON_KEY` in `.env`, run `npm run config` |
| Cloud save fails | Check RLS policies, unique `user_id`, and browser console for errors |
| Styles not on production | Confirm Vercel env vars and that deploy ran `npm run build` |
